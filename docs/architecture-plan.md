# 极简任务管理-番茄钟系统架构与开发计划

日期：2026-06-13

补充：2.0 跨平台低内存架构方向已存档到 `docs/v2-architecture-decision.md`。本文件仍记录 1.x Windows 本地优先版本的架构和开发计划。

## 1. 目标和边界

目标是在 Windows 10 上实现一款极低资源消耗、可长期置顶的极简任务管理和番茄钟工具。它应当像一个桌面常驻仪表盘，而不是一个完整知识管理或团队协作系统。

核心要求：

- 常驻桌面，支持始终置顶，窗口小、轻、稳定。
- 尽量使用 Windows 自带能力，避免 Electron、浏览器壳、大型数据库和复杂后台服务。
- 主功能块包括任务清单、今日待办、已结束、番茄钟、日期。
- 预留 Codex 自动化检查结果接入能力；如联动复杂，第一版可以先不做。
- 功能取舍参考成熟任务管理和番茄钟工具，但只保留低消耗场景真正需要的能力。

非目标：

- 不做跨平台客户端。
- 不做团队协作、评论、附件、富文本笔记。
- 不做云同步作为第一阶段能力。
- 不做复杂项目管理视图，例如甘特图、看板自动化、依赖关系图。

## 2. 成熟产品参考和可借鉴点

参考对象：

- Todoist：成熟任务管理产品，强项是收集箱、今日视图、项目、标签、过滤器、自然语言日期和提醒。
- TickTick：任务管理和番茄钟结合较紧，强项是日程、习惯、番茄专注统计和跨设备。
- Focus To-Do：典型的番茄钟加任务列表组合，适合作为轻量专注工具的参考。
- Pomofocus：极简网页版番茄钟，强项是启动成本低、任务与番茄周期结合清晰。
- Microsoft To Do / Windows 生态：可参考“我的一天”、本地通知、任务计划等偏系统集成的交互方式。

应借鉴的能力：

- 一个清晰的“今日”视图，而不是让用户每天从总清单中翻找。
- 待办收集和今日执行分离：待办事项是库存，今日待办是承诺。
- 番茄钟可和具体任务绑定；不强制绑定，以降低启动阻力。
- 完成项要可追溯，但默认折叠，避免干扰当天执行。
- 支持低摩擦录入：单行输入、鼠标优先操作、回车提交文字。
- 支持最小必要提醒：当前任务、剩余时间、今日数量、已完成数量。

不适合第一阶段照搬的能力：

- 多层项目、子任务、标签、过滤器、复杂查询。
- 账号系统、云同步、移动端同步。
- 丰富统计报表、周报、积分、成就。
- AI 自动重写任务、复杂优先级算法。
- 插件体系和主题市场。

## 3. 需求问题序列

本节维护需求不足和决策状态，便于后续逐条交互。状态含义：

- 已关闭：需求已经足够明确，进入设计和实现。
- 已后置：不是第一版阻塞项，保留接口或阶段计划。
- 待确认：建议先给出默认方案，后续可按使用感受调整。

1. 已关闭：任务清单、今日待办、已结束的关系

   你的回复合理。今日待办应作为任务清单的今日视图，而不是独立任务池。规则为：任务清单显示所有未完成任务；今日待办显示 `scheduledFor == 当前日期` 的未完成任务；已结束显示已完成历史。今日待办中直接添加的任务，也会同步写入任务清单并自动安排到今天。

2. 已关闭：完成任务后的数据保留

   你的回复合理。任务完成后从任务清单和今日待办的活动视图清理，进入已结束；保留原始创建日期、完成时间、番茄数量、来源等字段。这个问题已并入数据模型，不再作为开放问题。

3. 已后置：Codex 自动化联动

   你的回复合理。Codex 自动化联动不应阻塞第一版。当前只保留 `codex-inbox.json` 文件契约和阶段 5 计划；第一版先完成本地任务和番茄钟闭环。

4. 已关闭：番茄钟是否必须绑定任务

   你的回复合理。番茄钟不强制绑定任务。用户点击今日待办任务时弹出“启动番茄钟 / 完成 / 取消”；从任务启动番茄钟时，完成的番茄计入该任务；未选择任务时，记录为未绑定专注。

5. 已关闭：置顶窗口的信息架构

   你的回复合理。窗口采用小型置顶操作面板，字体略大于常规工具窗口。顶部固定入口为：任务清单、今日待办、番茄钟、其它、关闭。“其它”承载已结束、设置、数据检查等低频信息。

6. 已关闭：提醒策略

   你的回复合理。提醒采用组合策略并放入设置。第一版默认建议：提示音开启、窗口颜色变化开启、任务栏闪烁开启、系统通知关闭；系统通知依赖环境较多，作为可选项更稳。

7. 已关闭：可靠性和完备性检验

   你的回复合理。第一版需要纳入完备性检验，包括 JSON 校验、原子写入、备份、跨天行为、番茄中断、长期运行和数据损坏恢复。

8. 已关闭：快捷键策略

   你的回复合理。第一版不做全局快捷键，避免和其它工具冲突。键盘只负责文字输入和输入框内回车提交；主要操作使用鼠标点击和弹出菜单。

9. 已关闭：资源和成功标准

   由项目定义即可。第一版以低资源常驻为目标：空闲 CPU 接近 0%，常规状态低于 1%；内存目标低于 80 MB，理想低于 50 MB；启动时间低于 1 到 2 秒；连续运行 8 小时无明显内存增长或倒计时漂移。

10. 新增待确认：跨天未完成任务如何处理

   默认建议：昨日未完成任务仍留在任务清单中，但不会自动出现在今日待办。用户需要手动安排到今天。这样能避免每天自动滚入一堆旧任务，破坏“今日承诺”的含义。

11. 新增待确认：小窗口内任务排序规则

   默认建议：任务清单按创建时间升序；今日待办按安排到今日的时间升序；新添加任务排在底部；完成后立即移入已结束。第一版先不做拖拽排序，避免增加实现复杂度。

12. 新增待确认：关闭按钮的语义

   默认建议：顶部“关闭”表示退出程序，而不是最小化到托盘。原因是第一版追求简单可预期；托盘常驻可以放到常驻体验优化阶段。

13. 新增待确认：已结束记录保留周期

   默认建议：本地长期保留，不自动删除；后续提供按日期查看和 CSV 导出。数据量在个人任务场景下增长很慢，不需要第一版做清理策略。

## 4. 推荐技术架构

### 4.1 总体选择

推荐使用 Windows PowerShell 5.1 + .NET Framework WinForms 作为第一版实现。

理由：

- Windows 10 默认具备 Windows PowerShell 5.1 和 .NET Framework 运行环境，部署门槛低。
- WinForms 可直接设置窗口 TopMost，适合做小型常驻窗口。
- 单脚本或少量脚本即可运行，便于快速迭代。
- 资源占用远低于 Electron 类方案。
- 后续如果需要更精致 UI，可迁移到 WPF，但第一版不必提高复杂度。

备选方案：

- WPF/.NET Framework：UI 更灵活，样式更好，但开发复杂度略高。
- AutoHotkey v2：置顶和热键优秀，但不是 Windows 自带资源，需要额外安装。
- Native Win32/C++：资源最低，但开发和维护成本高，不适合第一阶段。
- Python/Tkinter：可行，但引入额外运行时，不符合“充分利用 Windows 自有资源”。

### 4.2 进程结构

第一阶段采用单进程：

- `TaskPomodoro.ps1`：启动脚本和主 UI。
- WinForms 主窗体：始终置顶，负责展示和交互。
- `System.Windows.Forms.Timer`：每秒或每 5 秒刷新番茄钟倒计时。
- 可选 `FileSystemWatcher` 或低频轮询：后续监听 Codex 自动化输出文件变化。
- 本地 JSON 文件：保存任务、今日计划、番茄记录和配置。

暂不引入后台服务。开机启动可后续通过 Windows 启动文件夹或任务计划程序实现。

### 4.3 模块划分

建议目录：

```text
task-pomodoro/
  TaskPomodoro.ps1
  config/
    settings.json
  data/
    tasks.json
    pomodoros.jsonl
    codex-inbox.json        # 阶段 5 可选
    backups/
  scripts/
    import-codex-result.ps1 # 阶段 5 可选
    backup-data.ps1
  docs/
    architecture-plan.md
```

逻辑模块：

- UI 层：窗口、按钮、列表、输入框、状态展示。
- TaskStore：读写任务 JSON，提供新增、完成、安排到今日、取消今日安排。
- PomodoroEngine：计时状态机，处理工作/休息/暂停/完成。
- CodexInbox：可选模块，读取 Codex 输出，去重、合并、生成候选今日待办。
- Reminder：按设置组合使用系统提示音、窗口闪烁、toast 或颜色变化提醒番茄结束。
- Settings：窗口位置、置顶、透明度、番茄时长、休息时长、提醒开关。

## 5. 数据模型

### 5.1 任务

```json
{
  "id": "20260613-153012-a7f3",
  "title": "整理 Codex 自动化输出格式",
  "source": "manual",
  "status": "todo",
  "createdAt": "2026-06-13T15:30:12+08:00",
  "scheduledFor": "2026-06-13",
  "completedAt": null,
  "pomodoroCount": 0,
  "priority": 2,
  "notes": "",
  "external": {
    "codexRunId": null,
    "fingerprint": null
  }
}
```

状态建议：

- `todo`：未完成任务，会出现在任务清单中。
- `done`：已完成。
- `archived`：归档，不在常规视图显示。

今日待办不是独立状态，而是派生视图：

- 今日待办：`status == "todo"` 且 `scheduledFor == 当前日期`。
- 任务清单：`status == "todo"` 的全部未完成任务。
- 已完成：`status == "done"` 的历史任务。

在今日待办视图直接新增任务时，系统创建一条普通任务，并将 `scheduledFor` 设置为当前日期。完成任务时，`status` 改为 `done`，记录 `completedAt`，保留 `createdAt` 和 `pomodoroCount`。

### 5.2 番茄记录

用 JSON Lines 追加写入，降低损坏风险：

```json
{"id":"pomo-20260613-160000","taskId":"20260613-153012-a7f3","startedAt":"2026-06-13T16:00:00+08:00","endedAt":"2026-06-13T16:25:00+08:00","plannedMinutes":25,"actualMinutes":25,"result":"completed"}
```

未绑定任务启动番茄钟时，`taskId` 允许为 `null`：

```json
{"id":"pomo-20260613-170000","taskId":null,"startedAt":"2026-06-13T17:00:00+08:00","endedAt":"2026-06-13T17:25:00+08:00","plannedMinutes":25,"actualMinutes":25,"result":"completed"}
```

`result` 可取：

- `completed`
- `interrupted`
- `skipped_break`

### 5.3 Codex 自动化输出

Codex 自动化作为阶段 5 以后的能力。若启用，建议每次检查后输出到固定文件：

```json
{
  "runId": "codex-20260613-090000",
  "generatedAt": "2026-06-13T09:00:00+08:00",
  "items": [
    {
      "title": "检查昨晚失败的自动化日志",
      "reason": "Codex 检查发现某任务连续失败 2 次",
      "priority": 1,
      "suggestedFor": "2026-06-13",
      "fingerprint": "sha256-of-stable-content",
      "sourceUrl": ""
    }
  ]
}
```

导入规则：

- 按 `fingerprint` 去重。
- `suggestedFor == today` 的项目进入“Codex 建议”区域。
- 不自动覆盖用户手动排序。
- 高优先级项目可按用户设置进入今日待办，并标记来源为 `codex`。
- 已完成任务即使再次出现，也只提示“已完成后又出现”，不自动重新打开。

## 6. UI 设计

窗口模式：

- 默认尺寸：约 320 x 420，可按实际字体和列表密度微调。
- 始终置顶：默认开启，可一键关闭。
- 支持折叠为迷你模式：只显示当前番茄、当前任务、今日剩余数量。
- 支持透明度：例如 85%-100%，默认 96%。
- 支持拖动位置并记忆。
- 字体略大于一般工具窗口，优先保证扫一眼能看清。

主界面：

- 顶部导航：任务清单、今日待办、番茄钟、其它、关闭。
- 任务清单：展示所有未完成任务，可安排到今日或完成。
- 今日待办：展示今日视图，可直接添加今日任务。
- 番茄钟：展示倒计时、开始、暂停、继续、停止。
- 其它：包含已结束、设置、数据检查、关于等低频入口。
- 已结束：展示完成记录，默认不占主界面空间。
- Codex 建议：后续接入时放在“其它”中，或以数量提示进入今日待办。

交互原则：

- 新增任务只需要输入文字后回车。
- 第一版鼠标优先，不设置全局快捷键，避免和其它工具冲突。
- 点击今日待办任务，弹出“启动番茄钟 / 完成 / 取消”三个选项。
- 点击任务清单任务，弹出“安排到今日 / 完成 / 取消”三个选项。
- 番茄钟可以无任务启动，记录为未绑定专注。
- 番茄结束后，若绑定了任务，则自动给该任务 `pomodoroCount + 1`。
- 完成任务后从任务清单和今日待办活动视图移除，进入已结束。

## 7. 资源预算

第一版建议验收指标：

- 空闲 CPU：接近 0%，常规状态低于 1%。
- 内存：目标低于 80 MB，理想低于 50 MB。
- 启动时间：低于 1 秒到 2 秒。
- 倒计时刷新：1 秒刷新即可；空闲时避免高频循环。
- 文件监听：仅在启用 Codex 联动时开启；失败时使用 30 秒低频轮询。
- 数据写入：用户操作后即时写入，番茄计时不每秒写入，只在状态变化时写入。
- 数据可靠性：写入前做 JSON 基本校验，写入时先写临时文件再替换。
- 长期运行：连续运行 8 小时不应出现明显内存增长或倒计时漂移。
- 完备性检验：覆盖新增、安排今日、完成、跨天、番茄完成、番茄中断、数据损坏恢复。

## 8. 开发计划

### 阶段 0：原型确认

产出：

- 单脚本 WinForms 窗口。
- 可以置顶、拖动、关闭。
- 显示顶部导航：任务清单、今日待办、番茄钟、其它、关闭。
- 显示日期、一个任务输入框、列表占位、一个倒计时。

验收：

- Windows 10 可直接通过 PowerShell 启动。
- 窗口可保持在其他应用上方。
- 空闲 CPU 接近 0%。

### 阶段 1：本地任务闭环

产出：

- `tasks.json` 读写。
- 新增任务、安排到今日、完成任务、查看已结束。
- 今日待办视图新增任务时自动写入任务清单并安排到今日。
- 记忆窗口位置和置顶设置。
- 简单备份。

验收：

- 关闭后重开数据不丢。
- 今日待办作为 `scheduledFor == today` 的视图能正确跨天区分。
- 完成后保留创建时间、完成时间和番茄数量。
- 数据文件损坏时能保留备份并提示。

### 阶段 2：番茄钟闭环

产出：

- 工作/短休息计时。
- 开始、暂停、继续、停止。
- 番茄完成写入 `pomodoros.jsonl`。
- 完成后按设置组合触发提示音、窗口颜色变化、任务栏闪烁或系统通知。
- 支持不绑定任务启动番茄钟。

验收：

- 绑定任务时，番茄结束能正确累加到任务。
- 未绑定任务时，番茄记录能正常保存且不污染任务统计。
- 暂停/中断不会错误计为完成。
- 长时间运行 8 小时无明显 CPU 异常。

### 阶段 3：完备性检验和设置

产出：

- 设置页：番茄时长、休息时长、透明度、提醒方式。
- 数据检查：JSON 格式校验、备份检查、异常提示。
- 跨天逻辑检查：今日视图随日期更新。
- 已结束视图：查看完成任务和番茄数量。

验收：

- 提醒方式可单独开启或组合开启。
- 数据异常时能提示并保留最近可用备份。
- 今日视图在跨天后不显示昨日未重新安排的任务。
- 已结束记录可按日期查看。

### 阶段 4：常驻体验优化

产出：

- 迷你模式。
- 开机启动配置脚本。
- 低干扰提醒策略。
- 更细的鼠标菜单和点击区域优化。

验收：

- 可长期覆盖在浏览器、编辑器、文档窗口上。
- 不遮挡时可快速折叠。
- 不需要打开主窗口也能看见当前番茄状态。

### 阶段 5：Codex 自动化联动

产出：

- 约定 `data/codex-inbox.json`。
- Codex 输出导入脚本。
- UI 显示 Codex 建议。
- 去重、优先级、来源标记。

验收：

- 新 Codex 结果出现后，UI 能在 30 秒内发现。
- 重复结果不会重复生成任务。
- 用户手动修改的今日待办不会被自动覆盖。

### 阶段 6：可选增强

候选能力：

- 每日复盘摘要。
- 简单统计：今日完成数、番茄数、连续专注时长。
- 任务搜索。
- CSV 导出。
- WPF UI 重写。
- 使用 Windows 任务计划程序定时触发备份或 Codex 检查。

## 9. 第一版最小可行范围

建议第一版只做以下能力：

- 置顶小窗。
- 日期显示。
- 手动新增任务。
- 顶部导航：任务清单、今日待办、番茄钟、其它、关闭。

## 10. 关键风险和应对

1. PowerShell WinForms UI 可维护性一般

   应对：第一版保持 UI 简单，业务逻辑拆成函数；当界面复杂到维护困难时再迁移到 WPF。

2. Codex 自动化输出格式不稳定

   应对：第一版不接入，只预留 JSON schema；等本地任务和番茄钟闭环稳定后再做。

3. 长期置顶打扰用户

   应对：提供迷你模式、透明度和临时取消置顶。

4. 数据文件损坏

   应对：写入前备份，写入时先写临时文件再替换；番茄记录用追加日志。

5. 任务自动导入造成今日列表污染

   应对：Codex 项默认进入建议区；只有高优先级或用户配置允许时才自动进入今日。

6. 纯鼠标操作可能降低高频录入效率

   应对：保留输入框回车提交，不做全局快捷键；后续如果确有需要，只增加应用窗口内快捷键。
