# 收集箱、执行记录与昨日接续计划

最后更新：2026-06-22

本轮把“新想法收集 -> 正式任务 -> 执行过程 -> 执行记录 -> 次日接续”做成轻量闭环。功能必须保持本地、低打扰和低资源：不做全局监控，不记录翻译原文/译文，不做复杂统计、游戏化成就、排行榜或 Codex 自动导入。

## 收集箱

`收集` 是任务前缓冲区，不是 Today，也不是第二套任务系统。收集项存放在独立 `data/inbox.json` 中，不参与番茄、今日结算、归档、任务排序或任务链接打开逻辑。用户在 `收集` 页输入新想法；双击或右键可转为正式任务；右键 `安排到今日` 会创建正式任务并安排到今天。

未来 Codex 自动化只能写入收集箱或等价候选区，不得直接写入 Today。

## 行为事件

`data/behavior-events.jsonl` 是内部 append-only 事件流，用于执行记录和后续轻量连续性功能。第一版只记录最小核心事件：任务创建、安排/取消今日、任务启动、番茄开始/完成/中断、任务完成/取消、收集新增/转任务、昨日接续是/否。

事件字段固定为 `id`、`at`、`type`、`taskId`、`sessionId`、`source`、`payload`。事件不替代 `tasks.json` 或 `pomodoros.jsonl`，也不是审计/监控系统。

## 执行记录

`其它` 中的 `已完成` 直接升级为 `执行记录`。视图以任务为核心聚合：上方显示已启动但未完成任务，按最近活跃时间倒序；下方显示已完成/已取消任务，按结束时间倒序。旧任务没有行为事件时，仍通过 `completedAt` 和 `pomodoroCount` 显示为历史记录。

执行细节弹窗显示人可读事件，不暴露 JSON。

## 昨日接续

前一天结算后，当天用户第一次关注本系统时，如果昨日 Today 有未完成任务，只显示一次接续卡片：

```text
昨日完成 X 个番茄
昨日未完成 Y 项
是否接续昨日任务？
```

按钮只有 `是` 和 `否`。选择 `是` 时保留所有昨日 Today 未完成任务，跳到任务页并提示 `请选择今日任务`。选择 `否` 时按原设定清空 Today，再跳到任务页并提示同样文案。

## 验收边界

- 收集箱和正式任务数据分离，转任务后才进入任务库。
- 行为事件只由 workflow/result event 边界追加，UI 控件事件不直接写日志。
- 执行记录不做 dashboard，不展示翻译内容，不读取全局窗口活动。
- 昨日接续不改变每日归档职责，不重复弹出，不强制用户处理。
## 恢复统计与执行统计补充

`恢复成功` 定义为：同一任务出现 `pomodoro_interrupted` 后，后续再次出现同任务 `pomodoro_started`。不要求重新开始后的番茄完成；切到其他任务开始不算这次恢复。

每日接续提示只展示昨日恢复成功次数，文案为：`昨日完成 X 个番茄 / 恢复成功 Z 次 / 昨日未完成 Y 项 / 是否接续昨日任务？`。这里不展示加权中断次数，避免把每日提示变成压力统计。

执行记录右键菜单的 `执行统计` 是低频现算摘要，不新增 timer、不写缓存。单任务统计只显示四行：`恢复统计 x/y`、`番茄统计 x/y`、`时间记录：启动 ...｜最近 ...`、`专注时长 ...`。

`恢复统计 x/y` 中，`x` 为恢复成功次数，`y` 为加权中断次数。每次中断找到后续同任务下一次开始：0-5 分钟内接续计 1，超过 5 分钟且 10 分钟内计 2，超过 10 分钟或未接续计 3。

`番茄统计 x/y` 中，`x` 为实际完成番茄数，`y` 为计划番茄数；未设置计划时显示 `-`。`时间记录` 只保留启动和最近：已完成任务的完成时间本质上就是最近推进时间，未完成任务不存在完成时间，不单列完成。

`执行统计` 对话框内放一个可点击 `?`，只展示口径说明，不把说明文字和统计正文混在一起。
