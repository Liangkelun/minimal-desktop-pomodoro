# 极简桌面番茄项目管理章程

版本：v0.1  
状态：草案  
最后更新：2026-06-20

## 1. 项目定位

极简桌面番茄不是完整任务管理器，而是一个低打扰的桌面任务提醒与番茄钟工具。

核心定位：

- 桌面常驻的当前任务提醒
- 今日任务执行清单
- 与今日任务关联的番茄钟
- 可穿透的虚化提醒模式

产品主线固定为：

```text
任务库 -> 今日承诺 -> 当前执行 -> 番茄专注 -> 完成任务 / 归档任务
```

新增功能必须优先服务这条主线。

## 2. 目标用户与场景

核心用户是高频使用电脑的个人工作者，包括写代码、写文档、研究、学习和项目推进的人。

典型场景：

- 从任务库挑选今日要做的任务
- 工作时只关注当前最重要的一两项
- 从今日任务启动番茄钟
- 需要沉浸工作时启用虚化模式，用透明背景文字层让任务提醒常驻但不阻碍其它窗口操作
- 完成则完成任务，不再推进则归档任务

本项目暂不服务团队协作、复杂项目管理、跨端同步和报表统计场景。

## 3. 产品原则

### 3.1 极简但不隐晦

界面应尽量少，但不能把基础操作藏到用户猜不到的地方。视觉极简不能换来操作复杂。

### 3.2 基础操作符合 Windows 习惯

基础操作应直接、常见、低学习成本：

- 左键选择
- 空白处左键清除选择
- 右键打开菜单
- 双击执行默认动作
- 边缘调整窗口大小
- `X` 关闭
- `-` 最小化，`□` 恢复
- 暂停后按钮变为继续
- 设置保存后即时生效，语言切换不需要重启

### 3.3 高阶操作可发现

高阶操作可以隐藏，但必须有规律，并能从帮助中找到：

- 拖动任务排序
- 顶部热区拖动窗口
- 输入序号插入任务
- 底部靠近时显示输入栏和导航
- 右键 `...` 承载低频操作

### 3.4 独特操作必须可解释、可退出

虚化模式、点击穿透、只保留 `△` 退出等独特交互必须在帮助中说明。虚化模式只能保留一个明确可靠的退出点；右键菜单只承载 `转换`、`翻译/停止翻译`、`翻译设置` 等低频入口，不扩展成复杂操作中心。

### 3.5 本地优先、低打扰

用户数据默认留在本地。帮助、关于、支持、治理等低频内容不能进入主流程。

## 4. 功能边界

### 当前核心功能

- 任务库
- 今日任务
- 番茄钟
- 完成任务
- 归档任务
- 拖动排序
- 任务编辑
- 虚化穿透提醒
- 今日任务 3 分钟破冰启动
- 轻量帮助
- 本地配置与本地数据

### 暂不推进

- 复杂统计报表
- 标签、项目、多层子任务
- 云同步和账号系统
- 团队协作
- 主题系统
- 重型赞助页或治理中心
- 更多隐藏手势

## 5. 交互词表

为降低认知成本，核心操作词汇应保持稳定：

- 安排到今日
- 放回任务
- 番茄钟
- 完成任务
- 编辑任务
- 归档任务
- 任务置顶
- 先做 3 分钟
- 时间提醒
- 截止时间

词表变更需要先评估语义、误解风险和与现有菜单的关系，不应随手替换。

## 6. 变更流程

### 6.1 意向先评估

用户提出意向式建议时，先评估和拆解，不直接开发。评估至少回答：

- 是否符合项目定位
- 是基础操作、高阶操作还是独特操作
- 是否会增加隐藏复杂度
- 是否影响虚化、低行数窗口、右键菜单或数据结构
- 是否需要帮助文案同步更新

### 6.2 小步提交

每次变更应尽量围绕一个明确目标，避免混入无关重构。高风险改动优先拆小。

### 6.3 数据保护

运行时数据、个人设置和日志不进入版本管理：

- `task-pomodoro/data/*.json`
- `task-pomodoro/data/*.jsonl`
- `task-pomodoro/config/settings.json`
- `task-pomodoro/launch.log`

任何测试都不能留下 `__selftest` 数据。

## 7. 验收标准

代码变更完成前至少满足：

- PowerShell 脚本可解析
- 相关功能经过手动或自检验证
- 自检不污染用户数据
- 菜单文案与实际行为一致
- 若改动隐藏交互或独特交互，帮助说明同步更新

推荐验证命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\task-pomodoro\scripts\Invoke-AutomatedChecks.ps1
```

如果应用实例正在运行并占用数据文件，应避免运行会写数据的自检，改用跳过自检的只读检查：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\task-pomodoro\scripts\Invoke-AutomatedChecks.ps1 -SkipSelfTest
```

## 8. Git 管理规范

### 8.1 版本管理范围

应纳入 Git：

- 应用脚本
- 启动脚本
- 文档
- 图标和音频资源
- 资源生成脚本

不纳入 Git：

- 个人任务数据
- 番茄记录
- 本地窗口位置和个人设置
- 日志和临时文件
- 原始对话、外部评估原文和附件素材；应先提炼到 `docs/` 再提交

### 8.2 提交规范

提交信息建议使用：

```text
type: summary
```

常用类型：

- `feat`: 新功能
- `fix`: 修复
- `ui`: 界面和交互调整
- `docs`: 文档
- `test`: 测试
- `chore`: 工程维护

示例：

```text
ui: simplify border appearance
docs: add project charter
fix: avoid selection during window drag
```

## 9. 下一阶段优先级

近期优先做稳定性，不急着扩功能：

1. 虚化模式稳定性：穿透、退出、透明度、异常恢复
2. 低行数窗口状态：1 行、2 行、10 行、底部显隐
3. 右键菜单语义和层级稳定
4. 拖动排序、窗口拖动、空白清选的冲突消除
5. 帮助说明保持轻量且准确
6. 模块边界继续收敛：业务模块返回结果对象，主脚本保持装配层职责

## 10. 治理原则

当前采用单维护者决策。外部建议先进入评估，不直接进入开发。

治理关注点：

- 用户数据安全
- 主流程低打扰
- 功能边界克制
- 可恢复、可退出、可理解
- 说明文档随交互变化同步维护

## 11. 免费开源与成本回收原则

本项目当前定位为免费开源项目。成本回收是项目可持续机制，不是产品主线。

允许的支持方式：

- 反馈真实使用问题
- 提交可验证的小改进建议
- 改进说明文档
- 分享使用经验
- 自愿赞助
- 定制开发或付费支持

约束原则：

- 不做广告
- 不强制登录
- 不把基础功能放入付费墙
- 不收集用户任务数据用于商业目的
- 不用弹窗、红点或主流程入口提示赞助
- 赞助通道只放在低频入口，例如帮助右键菜单或项目主页

如果未来发布公开版本，赞助或支持链接应放在 `关于/支持/治理` 一类低频区域，并且不得影响本地免费使用。

### 11.1 翻译增强支持者方案（后置）

翻译增强开发完成并稳定后，可以考虑加入轻量支持者机制。该机制应保持自愿、低打扰和可绕过，不做账号追踪、设备指纹、强绑定或自动扣费。

默认策略：

- 翻译增强提供 30 天本地试用。
- 试用结束后，用户可选择继续免费基础功能，或自愿支持以继续免打扰使用翻译增强。
- 试用状态只做本地轻量记录；删除重装可重新试用也接受，不为此引入追踪机制。
- 不自动续费扣款；所有付费都必须由用户主动发起。
- 付费入口只放在低频区域，例如翻译增强到期提示、帮助右键菜单或项目主页。

建议价格：

- 月度支持者：`￥6/月`，作为主推价格。
- 一次性支持：`￥39`，给不想订阅的用户一个干净选择。
- 年度支持可后置，参考 `￥48/年`，不放在主界面造成套餐复杂度。

权益边界：

- 基础任务管理、番茄钟、虚化提醒等核心能力保持免费。
- 支持者权益优先放在翻译增强体验，例如更大本地词典、自定义词典、翻译历史或免打扰继续使用。
- 不用弹窗、红点、倒计时压力文案或主流程打断诱导付费。
- 文案应坦诚说明：项目不做强限制，付费主要是支持维护和减少重复配置成本。

## 12. 2.0 架构备忘

2.0 方向已存档到 `docs/v2-architecture-decision.md`。核心原则是：统一业务核心，分平台实现低内存常驻壳，完整界面按需启动。

该方向不改变 1.x 的近期维护策略。1.x 继续优先稳定当前 Windows PowerShell WinForms 版本；2.0 用于后续跨平台和极低静止态内存目标。
