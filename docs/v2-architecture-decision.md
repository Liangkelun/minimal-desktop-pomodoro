# 2.0 架构决策存档

日期：2026-06-19

## 1. 决策摘要

2.0 采用“统一核心 + 分平台低内存常驻壳 + 按需完整界面”的架构方向。

本决策不要求立刻替换当前 PowerShell WinForms 版本。1.x 继续作为 Windows 可用版本维护；2.0 作为后续跨平台、低内存、可长期维护的演进方向存档。

核心判断：

- 如果只追求 Windows 快速迭代，当前 PowerShell WinForms 仍可继续维护。
- 如果以 Windows、macOS、Linux/Unix 主流桌面系统为终局，WPF 不是最终答案。
- 如果静止态和番茄运行态内存占用是硬约束，Avalonia、WPF、Electron、Tauri 都不应作为常驻核心首选。
- 2.0 应把业务规则统一，把强平台能力拆到各平台常驻壳。

## 2. 架构形态

```text
task-core/
  任务模型
  今日任务规则
  番茄状态机
  3 分钟破冰状态机
  设置归一化
  JSON 存储和迁移
  归档规则
  平台无关测试

task-shell-windows/
  极低内存常驻小窗
  置顶、透明、点击穿透
  托盘、通知、音频、开机启动
  Windows 原生窗口能力

task-shell-macos/
  AppKit 菜单栏或浮窗
  置顶、透明、通知、启动项
  macOS 原生窗口能力

task-shell-linux/
  X11 优先
  Wayland 能力降级
  托盘、通知、窗口管理适配

task-full/
  设置
  历史记录
  数据检查
  复杂编辑
  统计和增强功能
  按需启动，用完退出
```

## 3. 分平台边界

只分平台壳，不分业务核心。

必须统一：

- `tasks.json`、`settings.json`、`pomodoros.jsonl` 的数据语义和迁移策略。
- 任务库、今日任务、完成、归档、排序和 3 分钟破冰规则。
- 番茄状态机和记录规则。
- 核心设置字段含义。不同平台可以隐藏不支持项，但不能改变同一字段的语义。
- 核心测试用例。

允许分平台：

- 置顶、透明、点击穿透。
- 托盘、菜单栏、通知。
- 开机启动。
- 音频播放后端。
- 文件打开和外部链接打开。
- 窗口拖动、缩放、吸附和多显示器行为。

## 4. 技术候选

### 4.1 常驻壳优先候选

1. Rust + 平台原生 API

   适合统一核心和低内存常驻进程。Windows 可使用 `windows` crate 调用 Win32；macOS 和 Linux 壳可按平台封装。

2. Rust + Slint

   适合跨平台低内存 UI 原型。需要验证置顶、透明、穿透、托盘和平台通知能力。

3. C++ + FLTK / wxWidgets

   适合极低资源常驻桌面工具。开发体验和现代 UI 表达力弱于 Rust/Slint，但资源占用有优势。

### 4.2 完整界面候选

完整界面不承担长期常驻职责，可以接受较高内存：

- Avalonia + .NET：适合跨平台设置页、历史记录、复杂编辑和后续增强。
- Tauri：适合 Web UI 生态和较快构建完整管理界面。
- 当前 PowerShell 版：可在过渡期继续承担 Windows 完整界面。

## 5. 性能目标

2.0 常驻壳的目标不是功能最多，而是在用户进行 AI 辅助编程、写论文、多项目并行时不抢内存。

常驻壳目标：

- 静止态内存：目标低于 30 MB，优秀低于 20 MB。
- 番茄运行态 CPU：接近 0%，常规低于 1%。
- 番茄运行态内存：不因计时自然增长。
- 透明/穿透模式 CPU：目标低于 2%。
- 连续运行 8 小时无明显内存增长和倒计时漂移。

完整界面目标：

- 允许按需启动。
- 关闭后必须释放常驻壳以外的内存。
- 不得要求完整界面常驻后台。

## 6. 迁移顺序

1. 保持 1.x PowerShell WinForms 版本可用。
2. 抽出或重写平台无关核心，优先覆盖任务、今日、番茄、设置和数据迁移。
3. 先做 Windows 低内存常驻壳，因为当前真实内存压力主要发生在 Windows 工作流。
4. 常驻壳读取和写入现有 JSON 数据，避免破坏用户数据。
5. Windows 壳稳定后，再评估 macOS 壳。
6. Linux/Unix 壳后置，优先支持 X11；Wayland 下接受能力降级。
7. 完整界面按需启动，不进入常驻路径。

## 7. 原型验收

2.0 不直接全量重写。先做最小原型：

- 读取现有 `tasks.json` 和 `settings.json`。
- 显示今日任务列表。
- 支持置顶小窗。
- 支持番茄倒计时。
- 支持静止态和运行态内存测量。
- 支持关闭后数据不丢。

原型通过后才进入正式迁移。

## 8. 当前不做

- 不在 1.x 中引入跨平台复杂度。
- 不把 Electron 或浏览器壳作为常驻层。
- 不把 Avalonia/WPF 作为省内存方案。
- 不同时启动三平台开发。
- 不在核心层引入账号、云同步、团队协作或复杂项目管理。
