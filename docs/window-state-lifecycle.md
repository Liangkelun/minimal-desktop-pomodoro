# 窗口状态与生命周期

最后更新：2026-06-22

## 用途

本文件约束主窗口、虚化、翻译、设置对话框和单实例启动之间的状态关系。凡是涉及窗口跳位、字号变化、点击穿透、浮层、timer、listener 或设置保存的改动，都应先阅读本文件。

## 状态所有权

- 主窗口位置、宽高、置顶、当前视图、任务行数和任务字号当前分散在主窗口与设置模块；目标由 `WindowStateCoordinator` 统一拥有写入权。
- 虚化模式只拥有透明度、点击穿透、虚化视觉、虚化退出点和必要的虚化布局状态；它不得拥有翻译取词、词典、API 或浮层内容。
- 翻译模式只拥有 UIA/剪贴板监听、词典/API 查询、短释义浮层、详细释义面板和运行期缓存；它不得拥有主窗口位置、尺寸、任务字号或虚化布局。
- 翻译设置对话框只拥有翻译相关设置。它打开期间可以通过 `TranslationRuntime` 和 `WatermarkRuntime` facade 暂停翻译监听或点击穿透，关闭后必须按原状态恢复。
- 自测拥有临时数据目录。默认自测不得写入真实用户数据。
## 虚化与翻译正交关系

- 实化、虚化、翻译是可组合状态，不是单向包含关系。`实化 + 翻译` 和 `虚化 + 翻译` 都必须是合法状态。
- 启动翻译只启动 `TranslationRuntime`；不得调用进入虚化布局、折叠窗口或重算任务行数的流程。
- 进入或退出虚化只启动/停止 `WatermarkRuntime`；不得启动翻译监听、加载词典或调用翻译 API。
- 虚化状态可以影响翻译浮层的视觉风格和点击穿透策略，但必须通过窗口状态协调边界或受控通知传递。
- 翻译浮层位置不是主窗口位置；翻译字号不是任务字号。两者不得共享设置字段或保存路径。

## 主要状态

| 状态 | 允许行为 | 禁止行为 |
| --- | --- | --- |
| 实化普通状态 | 编辑任务、切换视图、启动番茄钟、进入虚化 | 加载翻译词典、启动翻译 timer、注册剪贴板监听 |
| 实化 + 翻译 | 保持实化窗口布局，启动取词和浮层 | 改主窗口位置、宽高、任务字号、当前视图 |
| 虚化普通状态 | 保持虚化视觉和点击穿透，允许退出虚化 | 加载翻译词典、启动翻译 timer |
| 虚化 + 翻译 | 在虚化视觉上叠加取词和浮层 | 重新计算主窗口布局、把翻译位置写入主窗口位置 |
| 翻译设置打开 | 通过 runtime facade 暂停监听或点击穿透，保存翻译设置 | 直接访问 timer/listener 私有变量，顺带保存主窗口位置/尺寸/字号 |
| 关闭、重启、自测 | 释放资源，必要时保存用户明确修改的设置 | 留下 timer、listener、浮层或写入真实自测数据 |

## 关键不变量

- 启动翻译不得调用会改变主窗口布局的虚化进入流程；它只能启动监听、查询和浮层。
- 停止翻译不得退出虚化；进入或退出虚化也不得启动或停止翻译。只有用户明确点击 `翻译/停止翻译` 或关闭应用统一清理时，才改变翻译生命周期。
- 翻译短释义浮层和详细释义面板不得抢焦点，不得阻断下方窗口操作。
- 翻译详细面板的位置不是主窗口位置。没有稳定设计前，不应持久化独立拖动位置；如后续加入拖动点，设置项也只能归属于翻译详细面板。
- `SetClickThrough`、窗口样式、置顶和透明度更新必须尽量幂等；timer 中不能反复写同一个窗口样式。
- 保存翻译设置时应保留当前主窗口布局。翻译字号和任务字号是不同设置，不能互相覆盖。
- 单实例启动如果发现已有窗口，应尝试聚焦已有窗口并退出新进程，避免弹出阻塞式“已在运行”对话框。
- 事件通知不得绕过窗口状态所有权；任何 handler 修改主窗口状态都必须走窗口状态协调边界。
- 从今日任务行启动绑定任务的番茄钟时，默认保留当前今日任务视图；任务行右侧倒计时是只读展示，不应写入当前视图或窗口尺寸。

## 常见故障模式

- 翻译启动复用虚化进入流程，会把实化窗口误转成虚化折叠布局，表现为跳位、变小或任务字号看似被改。
- 翻译 timer 中反复更新窗口样式或重排主窗口，会造成主界面闪烁和资源消耗。
- 翻译设置保存时写入主窗口位置、尺寸或任务字号，会让关闭设置后主界面跳位或字号异常。
- 翻译设置对话框直接停止/启动 UIA timer 或剪贴板 listener，会让 UI 层持有 runtime 资源所有权，后续容易出现恢复顺序错误。
- 自测直接写真实 `data/`，如果中途退出，会留下 `__selftest__` 或污染运行时数据。
- 取词浮层和主窗口共用定位设置，会把“详细翻译位置”误当成“主窗口位置”。

## 跳位/字号/闪烁排查顺序

遇到“开启翻译后主窗口跳位、任务字号变小、主界面闪烁或详细翻译位置污染主窗口位置”时，按下面顺序定位，不先做视觉层补丁：

1. 检查 `TranslationRuntime` 启动、暂停、恢复和停止路径，确认没有调用虚化进入/退出布局、窗口尺寸计算、任务字号应用、视图切换或通用设置保存。
2. 检查 `TranslationSurface`，确认短释义和详细面板只创建/定位浮层，不写主窗体 `Location`、`Size`、任务字号、任务行数或当前视图。
3. 检查翻译设置保存路径，确认走 `Save-TranslationSettings` 或保留窗口字段的 facade，不走会同步主窗口字段的通用保存路径。
4. 检查 timer/listener tick，确认只处理取词、查询和浮层显示；不得在 300ms 轮询里反复设置主窗口样式、重排主窗口或刷新任务视图。
5. 最后才检查视觉样式、透明度和浮层避让；这些只能影响翻译浮层，不应成为修复主窗口状态污染的入口。

## 手动冒烟

1. 启动应用，确认只有一个主窗口和一个主要 PowerShell 进程。
2. 在实化普通状态开启翻译，主窗口位置、宽高、任务字号、当前视图不变。
3. 在虚化普通状态开启翻译，虚化视觉不重新折叠或跳位。
4. 停止翻译后，UIA timer、可选剪贴板监听、浮层和词典缓存释放；虚化状态按用户原状态保留。
5. 打开并关闭翻译设置，主窗口位置、字号和视图不变。
6. 双击打开第二个快捷方式，应聚焦已有窗口或静默退出新进程，不应出现阻塞式“已在运行”弹窗。
7. 未启用剪贴板监听时，翻译流程不改变系统剪贴板；启用后也只响应用户自己复制进去的文本。
8. 在今日任务页启动绑定任务的番茄钟，应留在今日任务页并在该任务行右侧显示倒计时；独立番茄不应显示到任务行。

## 自动检查关注点

- 翻译生命周期测试应断言启动前后主窗口布局、任务字号和当前视图保持一致。
- 设置默认值和迁移测试应覆盖 `TaskFontSize` 与 `TranslationFontSize` 分离。
- 边界检查应禁止翻译模块写剪贴板、模拟复制或复制恢复。
- 边界检查应禁止翻译设置对话框直接访问翻译 timer、剪贴板 listener 和 runtime 私有变量。
- 自测默认应使用临时数据目录，并确认真实 `data/tasks.json` 不出现自测任务。
